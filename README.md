## Общая информация
Данный программный комплекс был разработан специально для защиты дипломной работы в **Компьютерной Академии ТОП** по специальности **«Разработка программного обеспечения»**. Проект представляет собой реализацию классической 2D-аркады с глубокой интеграцией механик управления ресурсами и событийной архитектурой.

## Статус проекта
На текущий момент программный продукт находится в стадии **демонстрационной версии (Demo)**. Архитектура проекта спроектирована с учетом дальнейшего масштабирования и расширения функциональных возможностей.

---

## Ключевые игровые механики

### 1. Система разрушаемых объектов
В проекте реализована многоуровневая система фрагментации астероидов. Объекты классифицированы на четыре типа: от «Больших» монолитов до «Крошечных» осколков. Уничтожение крупного объекта инициирует процедуру каскадного спавна дочерних элементов меньшего размера.

### 2. Маневрирование и энергия (Dodge Mechanic)
Реализована система экстренного уклонения (рывка), позволяющая мгновенно сменить позицию в пространстве. Активация данной способности потребляет ресурс **энергии**, что накладывает на пользователя обязательства по стратегическому контролю запасов мощности.

### 3. Нарративные ветвления
Игровой цикл включает в себя систему текстовых сценариев. В процессе прохождения пользователю предлагаются решения, определяющие дальнейшее развитие событий, что влияет на сложность и доступные ресурсы.

### 4. Комплекс активных способностей
* **Модификация лазерного залпа:** Увеличение плотности огня через веерный выпуск снарядов с расчетом углового смещения.
* **Энергетический барьер:** Генерация защитного поля, нивелирующего входящий урон.
* **Множитель эффективности:** Алгоритм кратного увеличения получаемых баллов за деструкцию вражеских единиц.

## Архитектура программного обеспечения
Проект разработан на языке C# в среде Unity с применением принципов объектно-ориентированного программирования:
* **Иерархия классов:** Базовый класс `Enemy` инкапсулирует фундаментальную логику здоровья и коллизий, а специализированные классы (например, `EasyEnemy`) реализуют уникальные паттерны атаки.
* **Событийная модель:** Использование `UnityEvent` для уведомления менеджеров о деструкции объектов, что минимизирует связанность кода.
* **Асинхронные операции:** Реализация спавна и циклов стрельбы через сопрограммы (Coroutines) для оптимизации нагрузки на основной поток.

## Управление
* **W, A, S, D / Стрелки** — Векторное перемещение.
* **ЛКМ / Пробел** — Инициация основного орудия.
* **Shift** — Пространственный рывок (Додж).
* **1, 2, 3** — Активация программных способностей.
